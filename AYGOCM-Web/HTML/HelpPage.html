1. System Requirements

Please ensure that you definitely match the minimal system requirements as shown below:

  • Display: a minimal resolution of 1024 x 768
  • Operation System: Windows Vista/7/8/8.1/10

    • (Windows XP not supported.) [Click here to see why][Click here to hide]
    Since AYGOCM requires your computer to have .NET Framework 4.5 installed, the program will not work in Windows XP as the latest .NET that XP supports is 4.0. We have a Java Edition available here!

  • Processor: 1.6GHz processor

    • (Optimized for both single and multicore processors)

  • Hard Disk Space: 50MB Free hard drive space (Installer files)
  • RAM: 1024MB (1GB)

    • Others/Prerequisites:

  • Microsoft .NET Framework 4.5
  • Internet connection (For update checking, news and FAQ)
  • Mouse and Keyboard

2. Downloading and Installing the program

Assuming you already have this program, but you cannot remember how to download it, or you're a new user and want to download and/or install this. Please follow this step by step method, and your program should be installed in no time!

  • Download the program from the website below:

    Anime Yu-Gi-Oh! Card Maker Download Page

  • If you have downloaded the ZIP version then do the following:
    1. Extract ALL files into a folder
    2. Install the itc-souvenir-lt-light.ttf font
    3. Launch the setup.exe file and proceed with the installation
    4. You can now launch the application from your desktop or start menu
  • If you have downloaded the EXE version then do the following:
    1. Launch the animeygocardmaker.exe installer and click Yes if required
    2. Proceed the installer by pressing Next and choose to install the program with or without the font
    3. Keep proceeding onwards and exit the installer when it's complete
    4. Another installation box will appear, just proceed on with it and it should install the application
    5. You can now launch the application from your desktop or start menu
  • If successful, you'll now have the application installed on your computer, if something went wrong, please provide a screenshot and send it to: aygocm@gmail.com

Updating the program

This one is basically the installation guide above, however take note of these things when updating:

  • The program must be closed during the installation
  • If using the ZIP version, make sure all files are extracted or the update will fail
  • If any installation error occurs, go here for possible solutions
  • You will not be able to install older versions of the program while a later version is already present

Using the Save Editor

This tutorial will show you how to use the Save Editor.

  • First, press the button on the program. (Located next to the and buttons)
  • Next, choose a saved card configuration and open it up with the file explorer
  • If the file loaded successfully, then all the empty spaces should get filled in, if this isn't the case, please refer to the guide.
  • Below you will find the values that can go in each field:

Levels:
  • All level types can hold up to a value between 1 and 13.
  • For normal levels, you can use value 14 to get a ? level.

NoMonster:
  • NoMonster has 2 values, 0 and 1. 0 meaning False, and 1 meaning True.
  • Value 0 indicates that the card is a monster.
  • Value 1 indicates that the card is not a monster.

Spell Card:
  • Spell Card has value True and False.
  • If set to True, the card will be seen as a Spell Card.
  • If set to False, the card will not be seen as a Spell Card.

Trap Card:
  • Trap Card has value True and False.
  • If set to True, the card will be seen as a Trap Card.
  • If set to False, the card will not be seen as a Trap Card.

XYZ Monster:
  • (This setting should only be used if you are using version 4.1 or below, however version 5.0+ will still be able to read this setting from older configurations)
  • XYZ Monster has 2 values, 0 and 1. 0 meaning False, and 1 meaning True.
  • Value 0 indicates that the card should use XYZ levels.
  • Value 1 indicates that the card should not use XYZ levels.

Level Type:
  • (This setting only applies to version 5.0 or newer and replaces "XYZ Monster" setting)
  • Level Type has 3 values: N, X and D.
  • Value N indicates that the card should use Normal levels.
  • Value X indicates that the card should use XYZ levels.
  • Value D indicates that the card should use Negative/Dark levels.

ATK/DEF Value:
  • These two fields specify the ATK/DEF points that the card should have.
  • Although you can type values that have more than 5 characters, the program will only display a maximum of 5 characters.

Attribute ID:
  • This field specifies the attribute that the card will have, by calling out its ID.
  • ID's go from 1 to 7.
  • Here are the following ID's for each attribute:
  • Earth - 1
  • Water - 2
  • Fire - 3
  • Wind - 4
  • Dark - 5
  • Light - 6
  • Divine - 7
  • Typing any other value will not be read by the program.

Card ID:
  • This field specifies the template that the card will use, by calling out its ID.
  • ID's go from 1 to 21.
  • The list is huge so we put it into a Spoiler:
  • Click to Show ID's
    • Normal - 1
    • Effect - 2
    • Ritual - 3
    • Fusion - 4
    • Synchro - 5
    • XYZ - 6
    • Winged Dragon of Ra - 7
    • Obelisk the Tormentor - 8
    • Slifer the Sky Dragon - 9
    • Spell - 10
    • Trap - 11
    • Legendary - 12
    • Normal 1-Box - 13
    • Effect 1-Box - 14
    • Alternative Ritual - 15
    • Alternative Fusion - 16
    • Token - 17
    • Ra with Egyptian Hieroglyphics - 18
    • Spell of Orichalcos - 19
    • Normal Pendulum - 20
    • Effect Pendulum - 21
    • ATK/DEF Token - 22
    • Dark Synchro - 23
    • Custom Normal - 24
    • Ritual Pendulum - 25
    • Fusion Pendulum - 26
    • Synchro Pendulum - 27
    • XYZ Pendulum - 28
    • Dark Synchro Pendulum - 29
    • LINK Monster - 30
    • Typing any other value will not be read by the program. Using a custom template will reset the Card ID back to 1

Custom Card:
  • Custom Card has value True and False.
  • If set to True, the card will be seen as a Custom Card.
  • If set to False, the card will not be seen as a Custom Card.
  • If Custom Card is True then the value for Card ID is irrelevant.
